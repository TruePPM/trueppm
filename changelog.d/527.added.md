Program Settings → Rollup KPIs is now wired to a real API (`GET|PATCH /api/v1/programs/{id}/rollup-config/`, ADR-0079). Program admins can toggle which of 10 health signals roll up to the program overview (Schedule, Risk, Cost groups) and choose the health aggregation policy (`worst` / `average` / `weighted_by_budget` / `task_weighted`). New programs are seeded methodology-aware defaults (waterfall, agile, or hybrid). All config changes are captured in the existing Program history records for audit.
