- **WebSocket event names for task collaboration** are now per-action past-tense (`comment_created`, `comment_updated`, `comment_deleted`, `attachment_created`, `attachment_deleted`) instead of `comment_changed` / `attachment_changed` with an `action` field. This matches the existing `task_created` / `project_updated` convention. Clients listening to the old combined events must update their handlers — no compatibility shim. Payload shape is unchanged otherwise (`id`, `task_id`, and `parent_id` for comments) (ADR-0075 §D, M-1).
