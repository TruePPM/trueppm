Hardened the `trueppm-scheduler` engine against adversarial input. Degenerate calendars (no working day, or holiday exceptions blanketing the schedule), out-of-range task durations and dependency lag, non-positive Monte Carlo run counts, cyclic summary-task hierarchies, and non-finite JSON literals (`NaN`/`Infinity`) are now rejected eagerly with a clear `InvalidScheduleInput`/`ValueError` instead of spinning the CPU for millions of iterations and raising an uncaught `OverflowError`/`RecursionError`. The same bounds are enforced at the API edge (`Task.duration`, `Dependency.lag`, `Calendar.working_days`) so a crafted project can no longer tie up the synchronous Monte Carlo request path, and the WASM scheduler uses checked date arithmetic so the same input can no longer panic the engine in the browser.
