Within-program resource contention API: `GET /api/v1/programs/{id}/resource-contention/` returns each resource's task spans aggregated across **every member project of a program**, each span tagged with its source project, so a caller can surface people over-allocated across sibling projects in overlapping windows. Read-only and Scheduler+ on the program (a new `IsProgramScheduler` permission). Overallocation detection stays client-side per ADR-0031. OSS, within-program visibility only — cross-program leveling remains Enterprise. Issue #1149.
